# 토스플레이스(Toss Place) 벤치마킹 분석

> 출처: https://tossplace.com / https://toss.tech/article/toss-pos-plugin
> 토스플레이스는 오프라인 매장의 주문·결제·매장 운영 통합 솔루션

---

## 토스플레이스 핵심 철학

1. **사용 교육 불필요**: 직관적 화면 구성, 간단한 가이드만으로 즉시 사용
2. **프로그램 우선**: 기계(하드웨어)가 아닌 소프트웨어 → 기존 태블릿·PC에 설치
3. **전 OS 지원**: Windows, Android, iOS, macOS 전부 지원
4. **모바일 확장**: 2024년부터 사장님도 고객도 스마트폰으로 주문·결제
5. **플러그인 생태계**: VSCode 익스텐션처럼 기능 확장 (앱스토어 방식)

---

## 주요 기능 목록

## 플러그인 아키텍처 (VSCode 방식)

### 핵심 개념
- 포스 = 코드 편집기 (VSCode)
- 플러그인 = 익스텐션 (ESLint, Prettier 등)
- 웹 워커로 플러그인 격리 → 플러그인 장애가 POS 본체에 영향 없음

### 플러그인이 접근 가능한 리소스
- 주문 (order.add)
- 결제 (payment)
- 결제수단 (paymentMethod.add)
- 테이블 (table)
- UI 팝업 (ui.openPopup - 바코드 입력 등)
- 메뉴 (menu)
- WebSocket / HTTP 통신

### 현재 실제 사용 중인 플러그인
- 테이블 원격 주문 서비스
- 배달사 연동

### 향후 확장 계획
- HTML 직접 삽입으로 커스텀 페이지 표시
- 온보딩 화면 제공
- 플러그인 스토어 (앱스토어 방식으로 점주가 직접 설치)

## UI/UX 원칙 (토스 디자인 DNA)

1. **최소 조작**: 핵심 동작은 탭 1~2번으로 완료
2. **큰 터치 타겟**: 손가락으로 쉽게 탭 가능한 크기
3. **명확한 상태 피드백**: 결제 완료·실패 즉각 시각적 표현
4. **정보 계층화**: 중요 정보는 크게, 부가 정보는 작게
5. **어두운 배경 없음**: 밝고 깔끔한 화이트 베이스
6. **색상 코드**: 토스 블루(#0064FF) 포인트, 중성 회색 배경
7. **카드 기반 레이아웃**: 각 항목을 카드 UI로 구분
8. **Bottom Sheet 패턴**: 상세 설정은 아래에서 올라오는 시트로

## 우리 앱에 반영할 핵심 인사이트

### UI/UX
- 메인 화면: 메뉴 그리드(카테고리별) + 오른쪽 주문서 패널
- 결제 화면: 큰 숫자 표시 + 결제 수단 큰 버튼들
- 듀얼 모드: 스태프 뷰 ↔ 고객 뷰 전환
- 하단 네비게이션: 주문 / 예약 / 고객 / 리포트 / 설정

### 기술
- Electron 기반 (웹뷰 위의 JavaScript) → 우리는 Flutter
- 웹 워커 격리 → 우리는 Isolate (Flutter)
- 플러그인 SDK → Phase 후반에 구현

### 운영
- 하드웨어 없이 소프트웨어만으로 동작
- 기존 iPad에 앱 설치만으로 즉시 사용
- 프린터·영수증 설정 분리
