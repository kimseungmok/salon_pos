# UI/UX 전면 재설계 + DB 보완 — 일본 살롱 POS

> 작성일: 2026-06-06  
> 기준: 토스포스 스크린샷 25장 + Fresha Academy 42개 영상 + 살롱보드 분석  
> 목표: 토스포스 수준의 완성도 + 일본 살롱 현지화

---

## 1. 전체 네비게이션 구조 재설계

### 현재 → 변경
```
현재: POS / 予約 / 顧客 / レポート / 在庫 / スタッフ / 設定  (7탭)
변경: POS / 予約 / 顧客 / レポート / 設定  (5탭, 재고·스태프는 설정 하위)
```

### 이유
- iPad 화면에서 하단 탭 7개는 아이콘이 너무 작고 혼잡
- 토스포스: 좌측 사이드바 + 메인 콘텐츠 구조 사용
- 살롱에서 실제 빈번하게 사용하는 화면: POS, 예약, 고객 (3개)
- 재고·스태프는 관리자 영역 → 설정 하위로 이동

### 최종 탭 구조
```
BottomNav 5개:
① POS (会計)        — 판매/결제 메인
② 予約 (Calendar)   — 예약 캘린더
③ 顧客 (Clients)    — 고객 목록/검색
④ レポート (Reports) — 매출/분석
⑤ 設定 (Settings)   — 전체 설정 허브
```

#### 예약 블록 디자인
```
┌──────────────────┐
│ ● confirmed      │  ← 상태 컬러 도트
│ 田中 様           │  ← 고객명 (굵게)
│ カット+カラー     │  ← 시술명
│ 10:00-12:00      │  ← 시간
│ [⚠] [알러지]     │  ← 경고 아이콘 (알러지/주의고객)
└──────────────────┘
```

상태별 컬러:
- `pending` → 황색 (未確認)
- `confirmed` → 파랑 (確認済)
- `in_progress` → 초록 (施術中)
- `processing` → 초록 반투명 줄무늬 (発色中 — Processing Time)
- `completed` → 회색 (完了)
- `no_show` → 빨강 (無断キャンセル)

#### Processing Time 시각화 (Fresha 핵심 기능)
- `bufferMin` → `processingMin` 으로 컬럼 분리
- 발색 중 블록: 원래 블록 뒤에 반투명 줄무늬 블록 연결
- 해당 시간에 다른 고객 예약 가능 (겹침 허용 표시)
- 캘린더에서 "발색 대기 중" 텍스트 표시

#### 예약 상세 슬라이드업 시트
```
┌──────────────────────────────────┐
│  ■ ● 確認済                     │  ← 상태 뱃지
```

#### 예약 생성 폼 (개선)
- **반복 예약**: 繰り返し 토글 → 毎週/隔週/毎月 선택

- **그룹 예약**: グループ予約 탭 → 여러 고객 동시 추가

## 6. 화면 전환 흐름 (UX 플로우)

### POS 핵심 플로우
```
POS메인 → [고객 지정] → [시술 선택] → [할인/포인트] → 결제화면
→ [분할결제] → 결제확인 → 완료화면 → [영수증/LINE발송]
→ 새 주문
```

### 예약 → 결제 연동 플로우 (토스포스·Fresha 공통)
```
캘린더 → 예약 탭 → [来店確認] → [会計へ]
→ POS 결제화면 (고객·시술 자동 입력됨)
→ 결제 완료 → 예약 상태 '完了' 자동 변경
→ カルテ 자동 생성 유도 팝업
```

### 고객 추가 후 즉시 예약 플로우
```
고객 목록 → [+ 顧客追加] → 기본정보 입력 → 저장
→ "予約を追加しますか?" 다이얼로그
→ 예약 폼 (고객 자동 입력)
→ 캘린더 반영
```

## 7. iPad Air 2 최적화 포인트

- **해상도**: 2048×1536 (264ppi) — 고밀도 이미지 @2x 준비
- **화면 크기**: 9.7인치 — 가로 모드 기준 설계
- **iOS 15.x 최대**: Material 3 일부 미지원 컴포넌트 주의
- **메모리**: 2GB RAM — 이미지 캐싱 최소화 (`cached_network_image` 활용)
- **스토리지**: SQLite WAL 모드 유지
- **터치 타겟**: 최소 44px (Apple HIG 기준) — 버튼 충분히 크게
- **키보드**: 일본어 IME 대응 — `textInputAction`, `focusNode` 체계적 관리
