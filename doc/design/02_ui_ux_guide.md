# UI/UX 가이드

> 토스플레이스 디자인 DNA 기반, 살롱 전용 커스터마이징

---

## 디자인 원칙

1. **교육 없이 바로 사용** — 처음 켜는 사람도 5분 내 주문 완료
2. **최소 탭** — 핵심 동작은 1~2탭으로 완료
3. **크고 명확한 터치 타겟** — 최소 44×44pt (iPad 손가락 조작 최적화)
4. **즉각적 피드백** — 모든 액션에 즉각 시각·햅틱 응답
5. **정보 계층화** — 가장 중요한 정보를 가장 크게
6. **오류 방지 우선** — 실수할 수 없는 구조 설계

## 컴포넌트 라이브러리

### 버튼
```
PrimaryButton    — 파란 배경, 흰 텍스트 (결제, 확인)
SecondaryButton  — 흰 배경, 파란 테두리 (취소, 뒤로)
DangerButton     — 빨간 배경 (삭제, 환불)
GhostButton      — 배경 없음 (부가 액션)
```

### 카드
```
MenuCard         — 메뉴 그리드 아이템 (이름, 가격, 썸네일)
CustomerCard     — 고객 리스트 아이템
AppointmentCard  — 예약 아이템 (시간, 고객, 시술, 스태프)
SaleCard         — 결제 내역 아이템
```

### 패널
```
OrderPanel       — 주문서 사이드 패널
BottomSheet      — 상세 설정, 결제 수단 선택
AlertDialog      — 확인·경고 다이얼로그
LoadingOverlay   — 결제 처리 중 전체 오버레이
```

### 입력
```
SalonTextField   — 한국어/일본어 최적화 텍스트 필드
NumPad           — 금액·수량 숫자 패드 (큰 버튼)
DateTimePicker   — 예약 날짜·시간 선택기
SearchBar        — 고객·메뉴 검색
```

## 애니메이션 가이드

```
Duration:
  빠름: 150ms  (버튼 탭, 상태 변경)
  보통: 250ms  (패널 슬라이드, 카드 등장)
  느림: 400ms  (페이지 전환)

Curve:
  easeOutCubic: 대부분의 UI 애니메이션
  spring:       물리 기반 (바운스 효과 최소화)

결제 완료:
  초록 체크 애니메이션 (500ms) → 자동으로 주문서 초기화
```

## 오프라인 상태 표시

```
온라인:  상태바 없음 (기본)
오프라인: 상단에 주황색 배너 "오프라인 모드 - 데이터는 로컬에 저장됩니다"
동기화 중: 파란 배너 "동기화 중..."
동기화 완료: 배너 자동 사라짐 (1초 후)
충돌: 빨간 배너 + 해결 버튼
```
